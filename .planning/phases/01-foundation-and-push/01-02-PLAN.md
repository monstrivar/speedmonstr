       - Return { user, loading, signIn, signOut }
       - Named export only: export function useAuth()

    2. Create app/supabase/schema.sql with the COMPLETE Phase 01 schema from RESEARCH.md:
       - organizations table (id uuid PK, name, slug unique, plan default 'vekst', created_at)
       - users table (id uuid PK refs auth.users, organization_id refs organizations, name, email, role check, department_id nullable, created_at)
       - departments table (id uuid PK, organization_id refs organizations, name, keywords text[], created_at)
       - leads table (id uuid PK, organization_id refs organizations, department_id nullable, first_name, last_name, email, phone, company, description, source default 'webhook', status check with Norwegian values, created_at)
       - push_tokens table (id uuid PK, user_id refs users, token, platform check, updated_at, unique user_id+platform)
       - Enable RLS on ALL tables
       - RLS policies:
         - organizations: SELECT where user's org matches
         - users: SELECT where same org
         - departments: SELECT where same org
         - leads: SELECT where same org
         - push_tokens: ALL where user_id = auth.uid()

       Add comments at top: "Run this in Supabase SQL Editor. Phase 01 schema."

       - Enable the pg_net extension (needed for database webhooks): `CREATE EXTENSION IF NOT EXISTS pg_net;`
       - Enable realtime for leads table: `ALTER PUBLICATION supabase_realtime ADD TABLE leads;`
