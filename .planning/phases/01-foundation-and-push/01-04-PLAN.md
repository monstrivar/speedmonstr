<task type="auto">
  <name>Task 2: Supabase Edge Function for APNs push sending</name>
  <files>
    app/supabase/functions/push-notification/index.ts
  </files>
  <action>
    Create app/supabase/functions/push-notification/index.ts as a Deno Edge Function:

    1. Import serve from "https://deno.land/std/http/server.ts"
    2. Import createClient from "https://esm.sh/@supabase/supabase-js@2"

    3. Implement createApnsJwt(teamId, keyId, privateKeyPem) -> Promise<string>:
       - Build JWT header: { alg: "ES256", kid: keyId } -> base64url encode
       - Build JWT payload: { iss: teamId, iat: Math.floor(Date.now() / 1000) } -> base64url encode
       - Parse PEM private key: strip BEGIN/END markers, decode base64 to Uint8Array
       - Import key: crypto.subtle.importKey("pkcs8", binaryKey, { name: "ECDSA", namedCurve: "P-256" }, false, ["sign"])
       - Sign: crypto.subtle.sign({ name: "ECDSA", hash: "SHA-256" }, key, data)

       CRITICAL: Web Crypto returns ECDSA signature in IEEE P1363 format (raw r||s, 64 bytes) when using the ECDSA algorithm directly -- NOT DER format. This is different from Node.js crypto which returns DER. So NO DER-to-raw conversion is needed.
       Verify: the signature ArrayBuffer should be exactly 64 bytes. If it's longer, there's a problem.

       - base64url encode the signature (replace +/ with -_, strip =)
       - Return: `${header}.${payload}.${signature}`

    4. Implement the serve handler:
       - Parse request body: { type, table, record, schema, old_record } (Supabase webhook payload format)
       - Validate: type === "INSERT" and table === "leads"
       - Create Supabase admin client using SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY from Deno.env
       - Look up organization's push tokens:
         ```
         Get all users in the lead's organization_id, then get their push_tokens
         supabase.from("push_tokens").select("token, user_id, users!inner(organization_id)")
           .eq("users.organization_id", record.organization_id)
           .eq("platform", "ios")
         ```
         If the join syntax is tricky, do it in two queries:
         1. Get user_ids from users table where organization_id matches
         2. Get push_tokens where user_id in those IDs and platform = "ios"

       - Generate APNs JWT using env vars: APNS_TEAM_ID, APNS_KEY_ID, APNS_PRIVATE_KEY

       - Determine APNs host from env: APNS_ENVIRONMENT ("sandbox" or "production")
         - sandbox: https://api.sandbox.push.apple.com
         - production: https://api.push.apple.com
         - Default to sandbox for safety

       - For each token, send push via fetch:
         ```
         POST https://{apnsHost}/3/device/{token}
         Headers:
           authorization: bearer {jwt}
           apns-topic: no.monstr.app
           apns-push-type: alert
           apns-priority: "10"
         Body:
           {
             aps: {
               alert: {
                 title: "Ny henvendelse: {first_name} {last_name}",
                 body: record.description?.substring(0, 100) || "Ny lead mottatt"
               },
               sound: "default",
               badge: 1,
               category: "NEW_LEAD"
             },
             lead_id: record.id,
             phone: record.phone
           }
         ```

       - Log results (success/failure per token) for debugging
       - Return JSON response { success: true, sent: count }

    5. Add error handling:
       - Try/catch around entire handler
       - Return 400 for invalid payload
       - Return 500 with error message for unexpected failures
       - Log errors with console.error

    6. Add a comment block at the top explaining:
       - Required env vars: SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, APNS_TEAM_ID, APNS_KEY_ID, APNS_PRIVATE_KEY, APNS_ENVIRONMENT
       - Triggered by: Supabase database webhook on leads INSERT
       - Setup: Create webhook in Supabase dashboard pointing to this function
  </action>
