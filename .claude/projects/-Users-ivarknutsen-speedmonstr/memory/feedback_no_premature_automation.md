---
name: Don't automate prematurely
description: User prefers to prove workflows manually before building automations like Make.com scenarios
type: feedback
---

Don't build automations (Make.com scenarios, calendar integrations, etc.) until the manual workflow has been proven with real clients first.

**Why:** User wants to validate the process with the first few clients before investing in automation. "We can automate it after we have had a few clients."

**How to apply:** When suggesting workflow improvements, propose the manual version first. Only suggest automation after the user signals they have enough volume to justify it.
